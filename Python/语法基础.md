### 1. Python 语法特点：优雅与简约

Python 的核心哲学是 **“Less is More”**（少即是多）。与其他语言（如 C 或 Java）相比，Python 极大地减少了样板代码。

*   **简洁性**：没有繁琐的大括号 `{}` 和分号 `;`。
*   **可读性**：代码读起来像英语句子。
*   **强制缩进**：利用缩进来表示代码块的层级关系，这强制了代码的整洁。

**代码对比（打印大于 5 的数字）：**

```python
# Python 代码：干净、直观
number = 10
if number > 5:
    print('这个数字大于 5')
```

### 2. 标识符与关键字

**标识符 (Identifiers)** 是你给变量、函数、类起的名字。
**关键字 (Keywords)** 是 Python 语言保留的单词，具有特殊含义，**不能**用作标识符。

#### 标识符规则：
1.  由字母、数字、下划线 `_` 组成。
2.  **不能以数字开头**。
3.  区分大小写（`Age` 和 `age` 是不同的）。

#### 关键字列表（部分）：
`if`, `else`, `while`, `for`, `def`, `return`, `import`, `class`, `True`, `False`, `None` 等。

**代码演示：**

```python
import keyword

# 查看所有关键字
# print(keyword.kwlist) 

# 正确的标识符
user_name = 'Alice'
score_2023 = 98
_private_var = 'Hidden'

# 错误的标识符（会导致 SyntaxError）
# 2nd_place = 'Bob'   # 错误：数字开头
# my-name = 'Tom'     # 错误：包含连字符
# class = 'Math'      # 错误：使用了关键字 class
```

### 3. 变量命名规则与约定

虽然 Python 语法允许你用 `a = 1`，但为了代码的生命力，我们需要遵循 **PEP 8**（Python 官方代码风格指南）的命名约定。

*   **变量和函数**：使用 **蛇形命名法 (snake_case)**，全小写，单词间用下划线连接。
*   **常量**：使用 **全大写 (UPPER_CASE)**，单词间用下划线连接。
*   **类名**：使用 **大驼峰命名法 (PascalCase)**，首字母大写。

**代码演示：**

```python
# 好的命名：见名知意
student_age = 18        # 变量：蛇形
MAX_CONNECTIONS = 100   # 常量：全大写

# 糟糕的命名
a = 18                  # 不知道 a 代表什么
StudentAge = 18         # 不符合 Python 变量习惯（这是 Java/C# 风格）
```

### 4. 注释的使用方法

注释是写给“人”看的，解释代码的意图，Python 解释器会完全忽略它们。

*   **单行注释**：使用 `#` 开头。
*   **多行注释**：虽然 Python 没有专门的多行注释符，但通常使用三个单引号 `'''` 或三个双引号 `"""` 包裹。
*   **文档字符串 (Docstring)**：写在函数或模块的第一行，用于生成文档。

**代码演示：**

```python
# 这是一个单行注释，解释下面变量的含义
radius = 5.0 

'''
这是一个多行注释。
可以写很多行，
通常用于临时注释掉大段代码。
'''

def calculate_area(r):
    """
    计算圆的面积。
    
    参数:
    r (float): 圆的半径
    
    返回:
    float: 面积
    """
    pi = 3.14159
    return pi * (r ** 2)

# 查看函数的文档字符串
print(calculate_area.__doc__)
```

### 5. 代码缩进规则与最佳实践

这是 Python 最独特的特征。**缩进不仅仅是美观，它决定了代码的逻辑结构。**

*   **规则**：同一代码块必须拥有相同的缩进量。
*   **最佳实践**：**永远使用 4 个空格**作为一级缩进。不要使用 Tab 键（除非编辑器自动将 Tab 转换为空格），更不要混用 Tab 和空格。

**代码演示：**

```python
age = 20

if age >= 18:
    # 这里的缩进表示这行代码属于 if 语句块内部
    print('成年人')
    print('可以考驾照')
else:
    # else 块的缩进
    print('未成年')

print('程序结束') # 没有缩进，无论 if 结果如何都会执行
```

**常见错误：** `IndentationError: unexpected indent`（缩进错误）。

### 6. 语句分隔与续行

*   **分隔**：Python 通常使用**换行符**作为语句结束的标志。虽然可以使用分号 `;` 将多条语句写在一行，但**强烈不推荐**。
*   **续行**：如果一行代码太长（超过 79 字符），可以使用反斜杠 `\` 续行，或者利用圆括号 `()` 的隐式续行（推荐）。

**代码演示：**

```python
# 1. 语句分隔（标准写法）
x = 10
y = 20

# 不推荐写法
x = 10; y = 20; print(x + y)

# 2. 续行
# 方法 A：使用反斜杠（容易出错，不推荐）
total = 1 + 2 + 3 + \
        4 + 5

# 方法 B：使用圆括号（推荐，更加优雅）
total = (
    1 + 2 + 3 +
    4 + 5
)

# 字符串过长续行
sql_query = (
    "SELECT * "
    "FROM users "
    "WHERE id = 1"
)
```

### 7. 基本输入输出操作

这是程序与用户交互的桥梁。

*   **输出 `print()`**：将内容打印到控制台。
*   **输入 `input()`**：获取用户输入。**注意：`input()` 接收到的永远是字符串 (str) 类型。**

**代码演示：**

```python
# --- 输出 ---
name = 'Python'
version = 3.11

# 1. 基本打印
print('Hello World')

# 2. 格式化输出 (f-string) - 现代 Python 推荐写法
print(f'正在学习 {name}, 版本是 {version}')

# 3. 控制结束符 (默认是换行 \n，这里改为空格)
print('Loading', end='...')
print('Done') 
# 输出结果: Loading...Done

# --- 输入 ---
user_input = input('请输入你的年龄: ')

# 这里的 user_input 是字符串 '25'，不能直接做数学运算
print(f'你输入的类型是: {type(user_input)}')

# 需要进行类型转换
age = int(user_input)
print(f'明年你就 {age + 1} 岁了')
```

### 8. 代码风格规范 (PEP 8 基础)

写出“Pythonic”（地道）代码的关键。

1.  **缩进**：4 个空格。
2.  **行长**：每行不超过 79 个字符。
3.  **空行**：函数定义之间空两行，类的方法之间空一行。
4.  **空格**：
    *   运算符两侧加空格：`x = 1 + 2` (好) vs `x=1+2` (坏)。
    *   逗号后加空格：`print(a, b)`。
    *   函数参数默认值等号两侧**不**加空格：`def func(key=5):`。

### 9. 常见语法错误与调试技巧

作为初学者，遇到报错是常态。学会看报错信息（Traceback）是成长的捷径。

#### 常见错误类型：

1.  **SyntaxError (语法错误)**：代码格式不对，解释器无法理解。
    *   *例子*：`if x > 5` (忘了冒号)。
2.  **IndentationError (缩进错误)**：空格数量不对。
3.  **NameError (命名错误)**：使用了未定义的变量。
    *   *例子*：`print(a)` (a 之前没赋值)。
4.  **TypeError (类型错误)**：对错误的数据类型进行了操作。
    *   *例子*：`'5' + 1` (字符串加数字)。

**调试代码演示：**

```python
def divide_number(num):
    try:
        # 尝试执行的代码
        result = 100 / num
        print(f'结果是: {result}')
        
    except ZeroDivisionError:
        # 捕获除以零的错误
        print('错误：不能除以零！')
        
    except TypeError:
        # 捕获类型错误
        print('错误：请输入数字！')

# 测试
divide_number(10)   # 正常
divide_number(0)    # 触发 ZeroDivisionError
divide_number('a')  # 触发 TypeError
```

### 总结

Python 的语法设计旨在让你专注于**解决问题**，而不是与复杂的语法规则做斗争。
1.  **缩进**是 Python 的灵魂，请务必严谨。
2.  **变量命名**要清晰，像写文章一样写代码。
3.  遇到错误不要慌，**Traceback** 会告诉你错误发生的行数和原因。
