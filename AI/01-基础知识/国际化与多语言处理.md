# 写作提示 —— 国际化与多语言处理

使用说明：
- 生成多语言输入/输出与检索的工程指南，兼顾术语与一致性。

生成目标：
- 总结分词差异（中日韩等）与对 Token/成本的影响；多语种 Embedding 与检索策略。
- 设计术语表（Glossary）、翻译与一致性机制；评估与报表。
- 提供多语言 RAG 的注意事项与示例。

大纲建议：
1. 分词与 Token 差异（长度与成本）
2. 多语种 Embedding 与检索（跨语言相似度）
3. 术语与一致性（Glossary、风格与禁用词）
4. 多语言评估（任务定义、指标与数据集）
5. UI 与体验（语言切换、方向性、字符集）
6. 合规与隐私（跨境与第三方）

输出格式要求：
- Markdown；附多语言检索示例与术语表模板。
- 给出评估与报表建议。

质量检查清单：
- 检索与输出一致；术语规范与风格统一。
- 评估可执行；指标明确。
- 兼顾合规与用户体验。
