# 写作提示 —— WASM 与性能

使用说明：
- 介绍 WebAssembly（WASM）在前端 AI/推理中的作用与性能优化手段。

生成目标：
- 解释 WASM 的运行模型、与 JS 的交互成本、SIMD 与多线程支持状况。
- 给出将原生/跨平台库通过 WASM 在浏览器运行的最小示例。
- 总结性能优化手段与常见坑（内存、数据拷贝、调用频率）。

大纲建议：
1. WASM 基本原理与运行时（栈、线性内存、宿主交互）
2. 与 JS 的边界与成本（调用边界、数据传递）
3. SIMD/线程支持与限制（浏览器差异）
4. 框架与工具链（Emscripten、Rust/wasm-pack、ONNX Runtime Web）
5. 最小示例（加载 WASM 模块、执行计算、返回结果）
6. 性能与内存优化建议（拷贝、缓冲区、批处理）

输出格式要求：
- Markdown；提供构建与运行的步骤说明。
- 给出性能对比或度量方法建议。

质量检查清单：
- 示例可运行或易于运行；工具链步骤清楚。
- 明确与 JS 交互的瓶颈与优化方案。
- 指出浏览器兼容性与降级策略。
