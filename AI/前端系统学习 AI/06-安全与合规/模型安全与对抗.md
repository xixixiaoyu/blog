# 写作提示 —— 模型安全与对抗

使用说明：
- 介绍 LLM 应用中的安全威胁与对抗场景，并提供防护策略。

生成目标：
- 解释常见攻击：提示注入、越权执行、数据外泄、越狱与不当内容生成。
- 给出防御方案：上下文隔离、输入清洗、策略提示、沙箱与资源限制。
- 提供评估与演练方法（红队测试、案例库与自动化检测）。

大纲建议：
1. 威胁模型与边界（攻击面与风险）
2. 常见攻击类型与案例（注入、越狱、外泄、对抗样本）
3. 防御措施（策略提示、工具权限、隔离与校验）
4. 观测与响应（告警、封禁、事后复盘）
5. 演练与评估（红队、自动化检测、黑名单/白名单）
6. 与合规的关系（内容审查与报告）

输出格式要求：
- Markdown；给出安全清单与演练流程建议。
- 附最小的输入校验或隔离示例。

质量检查清单：
- 威胁与防御说明具体可执行。
- 有评估与演练方法，不停留概念化。
- 与合规与内容审查章节能形成闭环。
