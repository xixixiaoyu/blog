# 目录写作提示（前端系统学习 AI）

使用说明：
- 将本提示复制到你的写作工具或 AI 助手，生成完整的课程目录页。
- 为每个章节添加仓库内的真实相对路径链接，保证可点击导航。
- 为每篇文章补充一句话简介与预计阅读时长（例如 10–20 分钟）。

生成目标：
- 输出一个清晰的课程目录，覆盖 00–07 所有模块与文章。
- 标注推荐阅读顺序、先修要求与建议时间分配，便于自学与查阅。

目录结构建议：
1. 课程总览与阅读指南（课程目标、适合人群、学习成果）
2. 模块索引（00–导读、01–基础知识、02–大语言模型、03–浏览器端 AI、04–服务端集成、05–工程实践、06–安全与合规、07–案例与模板）
3. 先修知识与环境准备（前端基础、Node 环境、包管理器、API Key 管理）
4. 学习路径建议（按初学者/进阶/实战导向的不同路径）
5. 更新日志与版本约定（语义化版本与内容变更说明）

输出格式要求：
- 使用 Markdown 编写；模块用二级标题，文章用三级标题。
- 每篇文章需包含：链接、一句话简介、难度（初级/中级/高级）、预计时长。
- 在文末附上“如何反馈问题与贡献内容”的说明。

质量检查清单：
- 所有链接均指向仓库内有效路径，点击可达。
- 模块、文件名与实际目录一致，命名统一。
- 阅读顺序合理，路径说明清晰，不互相冲突。
- 简介无术语歧义，关键术语统一。
- 有版本号或更新时间标注，便于维护。
