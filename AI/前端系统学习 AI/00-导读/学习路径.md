# 学习路径写作提示（前端系统学习 AI）

使用说明：
- 将本提示用于生成分层次的学习路径，覆盖本课程全部模块。
- 根据不同读者画像（前端初学者、具有后端经验、对 AI 有基础）提供差异化路线。

生成目标：
- 给出 3 条以上可执行的学习路径（初级/进阶/实战），每条路径包含阶段、目标与产出。
- 对每阶段给出先修知识、推荐阅读、练习任务与时间建议。

路径结构建议：
1. 总体目标与评估方式（完成后可实现的能力）
2. 初级路径（01 基础知识 → 02 LLM → 04 服务端集成 → 07 案例）
3. 进阶路径（03 浏览器端 AI → 05 工程实践 → 06 安全与合规）
4. 实战导向路径（从 07 案例出发，逐步补齐所需理论与工程能力）
5. 学习节奏建议（按周拆分，每周目标与练习）
6. 评估清单与里程碑（项目 DEMO、代码 Review、可量化指标）

输出格式要求：
- 使用 Markdown 编写；阶段用二级标题，任务用清单形式列出。
- 提供示例项目或练习仓库建议（如“实现一个流式聊天前端 + 代理后端”）。
- 用表述明确的可衡量目标（例如“实现 SSE 流式响应渲染”）。

质量检查清单：
- 路径覆盖面完整，顺序合理，有明确先修与依赖关系。
- 每阶段给出实践任务，避免纯理论堆砌。
- 有建议时长与产出要求，便于自检与复盘。
- 提供评估方法与达成标准，避免目标不明确。
