# 写作提示 —— 向量检索 + 重排序端到端示例

使用说明：
- 生成一个从索引到检索再到重排序与生成的端到端样例，突出工程细节。

生成目标：
- 数据管道：抽取 → 清洗 → 分块 → Embedding → 入库。
- 检索与重排序：向量检索 + BM25/交叉编码器重排；答案生成与引用。
- 评估与观测：覆盖率、相关性、答案质量与引用有效性。

大纲建议：
1. 数据与索引（分块、元数据、Embedding）
2. 检索与过滤（向量/关键词、混合检索）
3. 重排序（MMR/交叉编码器）
4. 生成与引用（去重、拼接、来源标注）
5. 评估与观测（指标与报表）

输出格式要求：
- Markdown；附完整接口与前端渲染代码片段。
- 给出评估与报表建议。

质量检查清单：
- 样例可运行或易于运行；接口清晰。
- 引用展示明确，避免误导。
- 有评估闭环与调优建议。
