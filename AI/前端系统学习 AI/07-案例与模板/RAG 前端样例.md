# 写作提示 —— RAG 前端样例

使用说明：
- 生成一个前端对接 RAG 后端的最小示例，展示检索、引用与答案渲染。

生成目标：
- 前端展示：问题输入、进度指示、答案与引用列表。
- 后端接口：检索结果、重排、生成与引用标注的 JSON 响应格式。
- 总结缓存、分页与过滤策略，提升体验与成本控制。

大纲建议：
1. 架构与数据流（前端 ↔ 后端）
2. 请求与响应格式（查询参数、返回字段、错误码）
3. UI 设计（答案、来源引用、折叠/展开、复制）
4. 体验优化（逐步渲染、滚动加载、去重与排序）
5. 缓存与成本（查询缓存、Embedding 复用）
6. 评估与观测（覆盖率、答案质量、引用有效性）

输出格式要求：
- Markdown；附最小接口规范与前端渲染代码片段。
- 标注依赖版本与运行步骤。

质量检查清单：
- 示例可运行或易于运行；接口清晰。
- 引用展示明确，避免误导。
- 有评估与观测闭环。
