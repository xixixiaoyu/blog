# 写作提示 —— Tokenizer 与 Embedding

使用说明：
- 生成对分词与向量嵌入的实用说明，强调与检索、RAG 的工程衔接。

生成目标：
- 解释 Tokenizer（BPE 等）与 Token 概念、对成本与长度限制的影响。
- 介绍 Embedding：维度、距离度量（cosine、L2）、常见用途（检索、相似度、聚类）。
- 提供前端或 Node 侧的示例代码（生成/查询 Embedding）。

大纲建议：
1. Token 与分词的必要性（长度限制与费用）
2. 常见分词算法与差异（BPE、WordPiece）
3. Embedding 的意义与用途（相似度搜索、聚类、重排序）
4. 距离度量与索引结构（平面 vs ANN；Faiss、Milvus、SQLite + pgvector 等）
5. 前端/Node 示例（生成、存储与查询向量）
6. 注意事项（维度选择、归一化、更新策略）

输出格式要求：
- Markdown；给出最小可运行示例或伪代码。
- 标注依赖版本与环境要求。

质量检查清单：
- 概念清楚，能与 RAG 章节顺畅衔接。
- 示例可运行或可快速改造成运行版本。
- 指出常见坑（分词不一致、维度不匹配、距离度量选择）。
