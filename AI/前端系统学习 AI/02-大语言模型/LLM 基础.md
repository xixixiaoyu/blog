# 写作提示 —— LLM 基础

使用说明：
- 生成对大语言模型（LLM）的入门介绍，强调工程直觉与应用边界。

生成目标：
- 解释 Transformer、Attention 的高层直觉，以及上下文窗口与长期记忆的关系。
- 介绍采样与解码（Greedy、Top-k、Top-p、温度）、函数调用、工具使用等能力。
- 说明局限与风险（幻觉、偏见、可靠性），以及工程上的缓解策略。

大纲建议：
1. LLM 的核心直觉（输入 Token → 概率分布 → 解码）
2. Transformer 高层结构与上下文窗口限制
3. 解码与采样策略（质量/多样性权衡）
4. 函数调用与工具使用（Function Calling、Tool/Agent）
5. 局限与风险（幻觉、事实性、指令遵循）
6. 可靠性提升（结构化输出、规划与校验、评测与回归）
7. 前端应用中的常见形态（聊天、摘要、改写、分类）

输出格式要求：
- Markdown；提供简单示意或伪代码说明。
- 配合前端场景给出采样参数的推荐范围与取舍。

质量检查清单：
- 直觉清晰，避开过度公式化。
- 工程实践建议具体可执行。
- 明确局限与风险，提出缓解方法。
