# 写作提示 —— Prompt 工程

使用说明：
- 生成可实操的 Prompt 工程指南，强调结构化输出与可评估性。

生成目标：
- 总结编写提示的原则：角色设定、任务拆解、约束与边界、示例驱动（Few-shot）。
- 介绍结构化输出（JSON/schema）、函数调用与可评估的 Rubric。
- 提供常见模板（总结、改写、分类、信息抽取、对话控制）。

大纲建议：
1. Prompt 的目标与边界（任务清晰与失败模式）
2. 角色/上下文/指令的配比（System/User/Assistant）
3. Few-shot 与思维链（何时使用、如何约束）
4. 结构化输出（JSON、schema、严格校验）
5. 函数调用与工具使用（参数校验、错误处理）
6. 模版库与复用策略（项目内统一约定）
7. 评估与回归（Rubric、自动判分、数据集维护）

输出格式要求：
- Markdown；同时给出模板与失败示例的对比。
- 附最小代码示例（调用 API 并解析结构化结果）。

质量检查清单：
- 模版清晰可复用；有边界与防御性说明。
- 结构化输出方案能在工程内落地。
- 给出评估与回归的可执行方法。
