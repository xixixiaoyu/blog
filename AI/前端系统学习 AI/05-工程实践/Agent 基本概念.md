# 写作提示 —— Agent 基本概念

使用说明：
- 解释 Agent 的核心思想与工程化形态，避免概念泛化与误解。

生成目标：
- 定义 Agent（目标、计划、工具使用、记忆、执行与评估）。
- 对比常见范式（ReAct、计划-执行、函数调用增强）。
- 给出最小可运行的工具使用示例与失败恢复策略。

大纲建议：
1. Agent 的要素（目标、计划、环境、工具、记忆）
2. 执行与控制（步骤拆解、约束、中止与恢复）
3. 工具调用与参数校验（错误处理与回退）
4. 观测与评估（日志、评分、回归）
5. 可靠性与安全（提示注入、防越权、资源限制）
6. 与应用架构的关系（何时需要 Agent、何时不需要）

输出格式要求：
- Markdown；附最小工具调用示例与失败恢复策略。
- 给出评估 Rubric 或度量建议。

质量检查清单：
- 定义清晰，避免把任意调用都称为 Agent。
- 示例可运行或易于运行；可靠性策略明确。
- 有评估与观测方法，避免黑箱。
