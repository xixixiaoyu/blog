# 写作提示 —— 安全提示与内容审查

使用说明：
- 生成在提示编写与响应处理中的安全策略与内容审查指南。

生成目标：
- 总结安全提示写法（约束边界、禁止事项、输出格式与语气要求）。
- 说明内容审查流程（分类、拦截、改写、上报与审计）。
- 提供前端 UI 与后端策略的协作模式。

大纲建议：
1. 安全提示的原则（明确边界与禁止事项）
2. 审查类别与策略（不当内容、隐私信息、违规话题）
3. 前后端协作流程（拦截、替换、提示与申诉）
4. 记录与审计（日志字段、保留策略）
5. 用户体验与透明度（提示说明、反馈渠道）
6. 与安全与合规章节的衔接（闭环）

输出格式要求：
- Markdown；提供审查规则示例与占位符模板。
- 给出 UI 文案与交互的建议。

质量检查清单：
- 策略明确可执行，避免含糊不清。
- 有日志与审计方案；与其他安全章节一致。
- 兼顾用户体验与透明度。
