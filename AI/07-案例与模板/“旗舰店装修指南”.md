# 写作提示 —— AI 工厂之“旗舰体验店”装修指南：React & Vue 标准化组件库

**核心类比**：将前端集成 AI 的过程，比作“AI 工厂”为自己的产品线，设计和装修“旗舰体验店”。本指南将为“体验店装修工程师”（前端工程师）提供一套标准化的“装修建材”（UI 组件）和“施工规范”（最佳实践），支持 React 和 Vue 两种主流“装修风格”。

**使用说明**：
- 本提示旨在将复杂的前端 AI 集成工作，转化为一套生动、直观的“店铺装修”流程。
- 你需要以“AI 工厂首席体验设计师”的口吻，为“前端装修团队”提供这份详尽的、跨框架的“装修指南”和“建材手册”。

**生成目标**：
- **统一“品牌形象”**：确立“AI 工厂”官方体验店的设计原则，强调安全、流畅、响应迅速的核心体验。
- **提供“标准化建材”**：为 React 和 Vue 分别提供一套核心的“预制装修组件”，包括“智能对话窗口”、“流式信息展示板”、“引用来源链接”等，确保“装修”效率和风格统一。
- **规范“施工流程”**：详细阐述从“后仓连接”（服务端代理）到“店内布局”（组件拆分）、“信息管理”（状态管理）和“顾客体验优化”（性能优化）的全套“施工规范”。
- **建立“安全生产”标准**：强调“电路安全”（密钥管理）和“消防通道”（错误处理）的重要性，确保“体验店”的安全稳定运营。

**大纲建议：《AI 工厂“旗舰体验店”装修指南》**

**第一部分：设计与安全规范**
1.  **第一章：“体验店”设计总则**
    *   引入类比，阐述“体验店”对于产品成功的重要性。
    *   核心设计原则：响应迅速、信息清晰、交互直观。

2.  **第二章：“安全施工”红线**
    *   **“电路安全”**：必须建立“店铺后仓”（服务端代理）来保管“钥匙”（API Key），严禁在“店面”（前端）直接存放。
    *   **“消防通道”**：设计完善的错误处理与重试机制，确保在“后仓”出现问题时，“店面”不会崩溃，并能给“顾客”清晰的提示。

**第二部分：“标准化建材”使用手册 (React & Vue)**
3.  **第三章：核心“建材”之“智能对话窗口”**
    *   **功能需求**：支持多轮对话、用户输入、加载状态、消息列表。
    *   **React 实现**：提供基于 Hooks 的 `ChatWindow` 组件代码。
    *   **Vue 实现**：提供基于 Composition API 的 `ChatWindow` 组件代码。

4.  **第四章：核心“建材”之“流式信息展示板”**
    *   **功能需求**：通过 SSE 或 WebSocket，将“中央厨房”传来的信息，以打字机效果实时展示给“顾客”。
    *   **React 实现**：展示如何使用 `useEffect` 监听事件并更新状态。
    *   **Vue 实现**：展示如何使用 `onMounted` 和 `ref` 监听事件并更新状态。

5.  **第五章：核心“建材”之“引用来源链接”**
    *   **功能需求**：当“产品”引用“知识库”内容时，清晰地展示来源链接，并支持点击跳转。
    *   **React/Vue 实现**：提供渲染引用列表的组件代码片段。

**第三部分：高级“装修”技巧**
6.  **第六章：“店内信息管理系统”（状态管理）**
    *   **React 方案**：使用 Context API 或 Zustand/Jotai 进行跨组件状态共享。
    *   **Vue 方案**：使用 Pinia/Vuex 进行集中的状态管理。

7.  **第七章：“顾客”体验优化**
    *   **“快速响应”**：利用缓存策略，减少对“后仓”的重复请求。
    *   **“减少等待”**：实现函数调用（Tools）的 UI 集成，在等待 LLM 响应时，先向用户展示工具执行的结果。
