# 写作提示 —— AI 工厂之“产品设计图纸”标准化模板库

**核心类比**：将 Prompt 模板合集，比作“AI 工厂”的“产品设计图纸标准化模板库”。这个模板库是工厂“产品研发部门”的核心资产，汇集了所有成功产品的“设计蓝图”（Prompts），供“产品设计师”（Prompt 工程师）快速复用、组合与创新，以加速新产品的开发流程。

**使用说明**：
- 本提示旨在将抽象的 Prompt 工程，转化为一套具象化的“产品设计”与“图纸管理”流程。
- 你需要以“AI 工厂首席产品设计师”的口吻，为新加入的“设计师”团队撰写这份“图纸库使用指南”。

**生成目标**：
- **建立“标准化设计”的理念**：强调使用“标准图纸”是保证产品质量、降低生产成本、提升研发效率的关键。
- **提供“图纸索引”**：将不同类型的 Prompt 模板，类比为不同系列的“标准产品设计图纸”，例如“S 系列（Summarization，摘要）”、“C 系列（Classification，分类）”、“E 系列（Extraction，抽取）”等，并为每个系列提供清晰的“产品定位”说明。
- **规范“图纸”的解读方式**：为每一份“标准图纸”提供统一的解读规范，包括其“设计目标”（使用场景）、“所需原材料规格”（输入格式）、“成品质量标准”（输出格式）、“成功案例与失败案例展示”（示例对比），以及最重要的“质量验收标准”（评估 Rubric）。
- **建立“图纸”的版本控制与评估体系**：引入“图纸版本号”的概念，并建立一套“设计评审”与“成品质量回归测试”流程，确保“图纸”的持续优化与质量稳定。

**大纲建议：《AI 工厂“产品设计图纸”标准化模板库使用指南》**

1.  **第一章：欢迎来到“产品设计中心”**
    *   引入类比，强调“设计图纸”（Prompt）是定义一个“AI 产品”灵魂的核心。
    *   阐述“标准化”的重要性：没有规矩，不成方圆。

2.  **第二章：“图纸档案室”导览（模板清单与分类）**
    *   **S 系列：信息浓缩大师**（总结类模板）
    *   **R 系列：写作风格大师**（改写类模板）
    *   **C 系列：智能分拣专家**（分类类模板）
    *   **E 系列：精准信息捕手**（抽取类模板）
    *   **J 系列：结构化数据生成器**（JSON 输出类模板）
    *   **T 系列：任务执行机器人**（函数调用类模板）
    *   *...（可继续扩充）...*

3.  **第三章：如何解读一份“标准设计图纸”**
    *   **图纸编号与版本**：`[系列]-[编号]-v[版本号]`，例如 `S-101-v1.2`。
    *   **产品定位**：该“图纸”适用于什么样的“市场需求”。
    *   **原材料规格**：对输入的文本有什么要求（长度、格式、内容）。
    *   **成品质量标准**：输出的格式、关键信息点和语言风格。
    *   **设计案例展示**：一个成功的“产品”示例和一个因“不按图施工”导致的“残次品”示例。
    *   **质量验收标准（Rubric）**：一套可量化的评分标准，用于评估“成品”是否合格。

4.  **第四章：“设计评审”与“质量回归测试”流程**
    *   **新“图纸”的设计与提交**：如何从零开始设计一份新的“图纸”，以及将其提交到“档案室”的审核流程。
    *   **“图纸”的优化与版本迭代**：当发现“图纸”有缺陷时，如何提出修改建议并发布新版本。
    *   **自动化“质检流水线”**：建立一套基于评估标准的自动化测试脚本，确保每次“图纸”变更都不会导致“产品质量”下降。

**质量检查清单（“指南”审批标准）**：
- **类比是否生动且贯穿始终？**：读者是否能完全沉浸在“产品设计师”的角色中？
- **模板库是否结构清晰、易于查询？**：“图纸”的分类和编号是否科学、合理？
- **“图纸”的解读规范是否完整？**：是否包含了从设计、输入、输出到评估的全流程要素？
- **流程是否具有可操作性？**：版本控制和质量保证的流程是否清晰，并能在团队中实际推行？
