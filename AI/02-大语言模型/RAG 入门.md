# 写作提示 —— RAG 入门

使用说明：
- 生成检索增强生成（RAG）的入门文章，强调架构、实现与评估。

生成目标：
- 解释 RAG 的组成：索引、检索、重排序、生成、引用。
- 介绍分块与切片策略、Embedding 选择与向量库选型。
- 给出最小端到端示例（构建索引 → 检索 → 生成 → 引用标注）。

大纲建议：
1. RAG 的目标与适用场景（事实性提升、资料问答）
2. 架构图与数据流（索引构建、检索、重排、生成）
3. 分块与切片（大小、重叠、分段策略）
4. 向量库与索引（本地/云、精度/性能权衡）
5. 生成与引用（答案拼接、来源标注、去重与排序）
6. 质量与评估（检索率、覆盖率、答案质量、引用有效性）
7. 缓存与成本优化（查询缓存、Embedding 复用）

输出格式要求：
- Markdown；提供端到端最小示例与架构图文字说明。
- 指明评估指标与实验记录方式。

质量检查清单：
- 架构清晰，流程可执行；示例可运行或易于运行。
- 明确评估方法与指标，避免凭直觉判断。
- 给出常见坑与防御策略（分块过大/过小、检索噪声）。
