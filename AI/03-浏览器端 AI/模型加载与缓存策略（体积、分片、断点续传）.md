# 写作提示 —— 模型加载与缓存策略（体积、分片、断点续传）

使用说明：
- 生成浏览器端加载模型的工程指南，优化体积与体验。

生成目标：
- 设计模型与权重的分片与断点续传；利用 HTTP Range、CDN 与缓存头优化下载。
- 提供渐进加载与占位渲染策略；处理版本更新与缓存失效。
- 说明内存占用、并发下载与错误恢复策略。

大纲建议：
1. 体积与分片（权重拆分、索引文件）
2. 断点续传与 Range 请求（CDN 与缓存）
3. 渐进加载与体验（占位、进度、降级）
4. 版本与缓存失效（ETag/Last-Modified、预热）
5. 并发与内存（下载并发、解压、释放）
6. 错误与恢复（重试、切片校验）

输出格式要求：
- Markdown；附最小加载与缓存示例代码。
- 给出版本管理与缓存策略建议。

质量检查清单：
- 加载稳定、体验良好；兼容低带宽与不稳定网络。
- 缓存与版本管理一致；避免旧模型污染。
- 有错误恢复与降级策略。
