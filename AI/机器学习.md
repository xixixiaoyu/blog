### 1. 机器学习是什么？

机器学习是一组让计算机从数据中“学习规律”、并据此进行预测或决策的方法。与手写规则不同，它通过优化模型参数来最小化误差，从而在未知样本上得到尽可能准确的结果。


### 2. 三大范式

- 监督学习：数据带标签，如 `图片 -> 猫/狗`、`房屋特征 -> 价格`。
- 无监督学习：数据无标签，主要做聚类与降维，如 `K-Means`、`PCA`。
- 强化学习：通过“状态-动作-奖励”的交互环境来学习策略，如游戏和机器人控制。


### 3. 一个标准流程

1. 问题定义：明确目标与评价指标（如 `RMSE`、`F1`）。
2. 数据准备：采集、清洗、划分训练/验证/测试集，处理缺失与异常值。
3. 特征工程：数值化、标准化、类别编码、构造新特征。
4. 模型选择：线性回归、逻辑回归、决策树、随机森林、XGBoost 等。
5. 训练与调参：网格搜索或贝叶斯优化，使用交叉验证评估。
6. 评估与上线：在测试集验证，部署并监控效果与漂移。


### 4. 过拟合与泛化

- 过拟合：模型把训练集“记住了”，在新数据上表现差。
- 缓解手段：更多数据、更简单的模型、正则化（`L1/L2`）、交叉验证、早停。


### 5. 常见算法速览

- 线性/逻辑回归：可解释性强、训练快，适合线性可分问题。
- 决策树与集成方法：能处理非线性与特征交互，随机森林稳定、XGBoost 常用于表格数据。
- 朴素贝叶斯：对文本分类常见，假设特征条件独立。
- 支持向量机：在中小数据集上效果好，但大规模时训练成本高。


### 6. 与深度学习的关系

深度学习是机器学习的一个分支，使用多层神经网络来自动学习复杂特征。在大规模数据与算力充足的场景（图像、语音、自然语言）中表现突出；但在特征相对稀疏、数据量有限的表格场景，传统机器学习（如 XGBoost）往往更实用。


### 7. 一个最小示例（伪代码）

```javascript
// 二分类：给定特征预测是否为正样本
const train = async (X, y) => {
  // 标准化
  const { XScaled, scaler } = standardize(X)
  // 训练一个逻辑回归
  const model = await fitLogisticRegression(XScaled, y)
  return { model, scaler }
}

const predict = ({ model, scaler }, XTest) => {
  const XScaled = scaler.transform(XTest)
  const prob = model.predictProba(XScaled)
  // 将概率转为标签
  return prob.map(p => (p >= 0.5 ? 1 : 0))
}
```


### 8. 工程注意事项

- 指标选择要贴近业务目标，避免只看单一指标。
- 保持训练/推理的一致性（特征处理、标准化参数不能漂移）。
- 加入监控与告警，关注数据分布漂移与模型老化。
