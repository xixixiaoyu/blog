# 写作提示 —— 多模态（语音与图像）集成

使用说明：
- 介绍在前端应用中集成语音与图像能力的实践，覆盖采集、上传、推理与呈现。

生成目标：
- 语音：麦克风采集、VAD、实时或离线 STT、TTS 播放与控制。
- 图像：选择/截图/摄像头采集、上传、解析（OCR/描述/问答）。
- 总结带宽、体积、隐私与权限的工程约束与优化。

大纲建议：
1. 能力边界与场景（语音对话、图片问答、多模态 RAG）
2. 采集与权限（麦克风/摄像头、浏览器 API）
3. 编码与传输（音频/图片格式、压缩与分片）
4. 推理与呈现（实时/离线、UI 控件与状态）
5. 性能与成本（带宽、延迟、缓存与复用）
6. 隐私与合规（权限提示、数据保留、匿名化）

输出格式要求：
- Markdown；给出最小采集与推理示例（语音/图像至少其一）。
- 标注依赖、权限与浏览器兼容性说明。

质量检查清单：
- 示例清晰可运行或易改造成运行版本。
- 明确权限与隐私处理，避免风险。
- 指出性能与体验权衡方案。
