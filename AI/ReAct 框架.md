### 1. ReAct 是什么？从名字说起

ReAct 这个名字是 **Reasoning** 和 **Acting** 的缩写，直译就是“推理”与“行动”。

这精准地概括了它的核心思想：**让大模型在解决问题的过程中，交替进行推理思考和采取行动。**

想象一下我们人类是如何解决一个复杂问题的，比如“规划一次从北京到上海的旅行，预算 3000 元，希望体验当地特色美食”。

你不会立刻说出所有答案。你的内心活动是这样的：

1. **思考**：我需要先查查机票或火车票的价格，这是大头。
2. **行动**：打开订票 App，搜索未来一周的票价。
3. **观察**：看到高铁二等座约 550 元，机票淡季约 600 元（含基建燃油费）。
4. **思考**：交通费大概 600 元，还剩 2400 元。接下来要查住宿和美食。
5. **行动**：打开酒店预订 App，搜索市中心的特色民宿。
6. **观察**：找到一家评分不错的民宿，每晚约 400 元。
7. **思考**：如果住两晚，就是 800 元。现在总花费 1400 元，还剩 1600 元用于美食和市内交通。钱应该够了。我需要再搜搜上海必吃的美食。
8. **行动**：打开美食 App，搜索“上海必吃榜单”。
9. **观察**：看到了小笼包、生煎、本帮菜等推荐。
10. **思考**：好了，信息收集得差不多了，可以制定一个详细的行程单了。

看到了吗？**思考 -> 行动 -> 观察 -> 再思考...** 这个循环，就是 ReAct 框架的精髓。



### 2. ReAct 的核心组件：思考、行动、观察

- ReAct 框架通过一个结构化的提示模板，引导大模型生成包含这三个关键环节的文本：
- **Thought (思考)**：这是模型的“内心独白”。在这里，模型会分析当前的情况，拆解问题，决定下一步该做什么。这是“推理”能力的体现。
- Action (行动)：基于思考的结果，模型选择并执行一个具体的“行动”。这个行动通常是调用一个外部工具，比如：
  - `Search(query)`：使用搜索引擎查找信息。
  - `Lookup(keyword)`：在维基百科等知识库中查找。
  - `Calculator(expression)`：使用计算器进行数学运算。
  - `CodeInterpreter(code)`：执行一段代码来分析数据或绘图。
- **Observation (观察)**：行动执行后，外部工具会返回一个结果。模型“观察”这个结果，并将其作为新的信息，用于下一轮的思考。

这个 **Thought -> Action -> Observation** 的循环会一直持续，直到模型认为问题已经解决，并给出最终的 **Answer (答案)**。



### 3. 为什么 ReAct 如此重要？它解决了什么痛点？

在 ReAct 出现之前，主要有两种范式：

1. **纯思维链**：模型通过生成一系列推理步骤来解决问题。这增强了模型的逻辑能力，但它完全依赖于模型内部存储的知识。如果遇到需要实时信息（如“今天天气如何”）或精确计算（如“12345 * 67890 等于多少”）的问题，模型要么无法回答，要么容易“一本正经地胡说八道”（产生幻觉）。
2. **纯工具调用**：模型可以理解指令并调用工具，但它缺乏自主规划能力。你必须明确告诉它每一步该做什么，它像一个被动的命令执行器，而不是一个主动的问题解决者。

**ReAct 的革命性在于，它将两者完美结合：**

- **用 Reasoning 指导 Acting**：模型的思考能力让它能像一个项目经理一样，分析复杂任务，制定计划，决定在何时调用哪个工具。
- **用 Acting 增强 Reasoning**：工具调用为模型提供了超越其内部知识的、准确的、实时的信息，这些信息反过来又让它的推理更加可靠和有根据。

简单来说，ReAct 将大模型从一个“博学的书呆子”变成了一个“知行合一的实干家”。



### 4. 一个简单的 ReAct 示例

让我们看一个技术化的例子，假设问题是：“莱昂纳多·迪卡普里奥主演的、上映年份晚于 2000 年的电影中，哪一部票房最高？”

一个基于 ReAct 的模型可能会这样生成：

Action: Search('Leonardo DiCaprio filmography')

Observation: 莱昂纳多·迪卡普里奥主演的电影包括：《泰坦尼克号》(1997)、《猫鼠游戏》(2002)、《无间行者》(2006)、《盗梦空间》(2010)、《被解救的姜戈》(2012)、《华尔街之狼》(2013)、《荒野猎人》(2015)、《不要抬头》(2021)等。

Thought: 好的，我得到了电影列表。现在我需要查询这些上映于 2000 年后的电影的全球票房，然后进行比较。我先从《猫鼠游戏》开始。

Action: Search('Catch Me If You Can box office')

Thought: 记录下来。接下来查《无间行者》。

Action: Search('The Departed box office')

Observation: 《无间行者》的全球票房约为 2.89 亿美元。

Action: Search('Inception box office')

Thought: 《盗梦空间》的票房超过了《猫鼠游戏》。我需要继续检查剩下的电影，以确保找到最高的。接下来查《华尔街之狼》。

### 5. 如何实现 ReAct？

ReAct 并不是对模型本身架构的改动，而是一种巧妙的 **提示工程** 技术。

开发者会设计一个包含几个完整 Thought-Action-Observation 循环的示例，放在提示的开头，然后让模型仿照这个模式来解决新问题。

```
你是一个通过推理和行动来解决问题的智能助手。请严格遵循以下格式。

Example:
Question: 姚明的身高是多少？
Thought: 我需要查找姚明的身高信息。
Action: Search('姚明身高')
Observation: 姚明的官方身高是 2.26 米（7英尺6英寸）。
Thought: 我已经找到了姚明的身高，可以回答问题了。
Answer: 姚明的官方身高是 2.26 米。

---
现在，请解决以下问题：
Question: [这里放入你的中文问题]
```

通过这种“少样本提示”，大模型就能学会并运用 ReAct 的模式。



### 总结

ReAct 框架是一个里程碑式的进步，它：

- **本质**：通过 **思考 -> 行动 -> 观察** 的循环，将大模型的推理能力与外部工具的执行能力相结合。
- **优势**：让模型能够解决需要实时信息、精确计算和多步规划的复杂问题，极大地扩展了其能力边界，并提高了回答的准确性。
- **实现**：主要依靠精巧的提示工程，而非改变模型底层结构。

它为我们构建更强大、更自主的 AI 智能体铺平了道路，是当前 AI Agent 领域最核心和最基础的技术范式之一。
