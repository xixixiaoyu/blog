这个问题触及了移动端开发中一个经典且微妙的视觉细节。要理解它，我们需要从屏幕成像的第一性原理说起。

## 问题本质

**移动端 1px 边框问题指的是**：在高清屏（Retina）设备上，使用 CSS 设置的 `border: 1px` 看起来比设计稿中预期的更粗、更“笨重”，失去了精致感。

其根源在于**CSS 像素**与**物理像素**之间的错位。

## 核心概念：设备像素比（DPR）

现代移动设备的屏幕普遍采用**高像素密度**技术。我们需要理解一个关键概念：

- **物理像素**：屏幕上的最小发光单元，是硬件层面的真实像素点
- **CSS 像素**：浏览器使用的抽象单位，是逻辑层面的，确保元素在不同设备上保持大致相同的视觉尺寸

**设备像素比（Device Pixel Ratio, DPR）** 是两者的换算关系：

```
DPR = 物理像素个数 / CSS 像素个数
```

- 普通屏（iPhone 3GS）：DPR = 1，1 个 CSS 像素 = 1 个物理像素
- 高清屏（iPhone 4/5/6）：DPR = 2，1 个 CSS 像素 = 2×2 个物理像素
- 超高清屏（iPhone X）：DPR = 3，1 个 CSS 像素 = 3×3 个物理像素

## 问题推导

当设计师在 2x 或 3x 设计稿上画出 1px 边框时，他们指的是**物理像素**级别的 1px。但 CSS 中的 `1px` 对应的是**CSS 像素**。

在 DPR = 2 的设备上：
- 要实现真正的 1 物理像素边框
- 需要 CSS 像素值为 `1 / DPR = 0.5px`
- 但 CSS 规范不支持小数像素（部分浏览器会四舍五入）

在 DPR = 3 的设备上：
- 需要 CSS 像素值为 `0.333px`

因此，当你写 `border: 1px solid #000` 时：
- **DPR=1**：✅ 1 物理像素，完美
- **DPR=2**：❌ 实际为 2 物理像素，看起来两倍粗
- **DPR=3**：❌ 实际为 3 物理像素，看起来三倍粗

## 具体表现

```css
/* 问题代码：在所有设备上都使用 1px */
.border-box {
  border: 1px solid #ddd;
}
```

在 DPR=2 的 iPhone SE 上，这条边框会占用 **2 个物理像素** 的高度，视觉上比设计稿粗一倍，破坏了精致的界面质感。

## 为什么这是个“问题”

这不仅仅是像素多少的问题，而是**视觉一致性**和**设计还原度**的问题。在高清屏普及的今天，用户已经习惯了发丝般精细的界面元素。粗边框会让产品显得廉价、不专业。
