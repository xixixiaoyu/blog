*   **开发者类比**：必须使用一个软件开发者非常熟悉的具体概念进行类比。拒绝宽泛的类比。
    *   **（优选）示例**：
        *   **Token**：就像编译器进行词法分析（Lexing）后产出的最小语言单元，是机器理解代码的原子。
        *   **Context Window**：就像 HTTP 请求的 Payload 大小限制，决定了你一次性最多能给服务器传多少数据。
        *   **Embedding**：就像一个高质量的、能理解语义的哈希函数，它能把相似的输入映射到相近的输出“指纹”。

*   **工程价值 (So What?)**：必须清晰、量化地阐述该术语对工程实践的直接影响。请至少从以下方面选择两点进行说明：
    *   **成本 (Cost)**：它如何影响你的 API 调用费用或服务器资源开销？（例如：Token 数量直接决定了 OpenAI API 的计费）
    *   **延迟 (Latency)**：它如何影响系统的响应时间？（例如：更大的 Context Window 或更复杂的 Prompt 会增加 LLM 的计算时间）
    *   **系统设计 (System Design)**：它如何影响你的架构决策？（例如：为了解决 Context Window 的限制，我们引入了 RAG 架构；为了实现高效的相似性搜索，我们选择了向量数据库）
    *   **用户体验 (UX)**：它最终如何体现在用户感知上？（例如：Hallucination 会严重损害用户对产品的信任度）

*   **代码/配置示例** (可选但强烈推荐): 提供一段简短的、说明性的代码（Python/JS 优先）或配置（如 JSON），来展示该术语在实际代码中的样子。
