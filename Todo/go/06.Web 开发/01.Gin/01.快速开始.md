# AI Prompt: Gin 框架快速入门教程

请你扮演一位热情且善于教学的 Go 语言导师，为零基础的 Go Web 开发者撰写一篇 Gin 框架的“快速开始”教程。

**文章目标：**
旨在引导读者完成从安装 Gin、编写并运行第一个 Web 服务器、到理解一个基本项目结构的整个流程。文章应尽可能地通俗易懂，步骤清晰，让读者在 10 分钟内成功运行自己的第一个 Gin 应用。

**核心内容：**

1.  **前置准备**
    *   明确指出读者需要预先安装好 Go 语言环境（并建议一个较新的版本，如 1.18+）。
    *   指导读者如何创建一个新的项目目录，并使用 `go mod init <module-name>` 初始化 Go 模块。

2.  **安装 Gin**
    *   提供清晰的 `go get` 命令来安装 Gin 框架。
    *   命令：`go get -u github.com/gin-gonic/gin`
    *   简单解释该命令的作用：下载并安装 Gin 包及其依赖。

3.  **编写你的第一个 Gin 应用：“Hello, World!”**
    *   **创建 `main.go` 文件**：指导读者在项目根目录下创建这个文件。
    *   **提供完整的代码**：
        ```go
        package main

        import "github.com/gin-gonic/gin"

        func main() {
            // 1. 创建一个默认的 Gin 引擎
            r := gin.Default()

            // 2. 定义一个路由和处理函数
            r.GET("/ping", func(c *gin.Context) {
                // 3. 返回一个 JSON 响应
                c.JSON(200, gin.H{
                    "message": "pong",
                })
            })

            // 4. 运行 HTTP 服务器，默认监听在 :8080 端口
            r.Run()
        }
        ```
    *   **逐行代码讲解**：
        1.  `gin.Default()`：解释它创建了一个带有 `Logger` 和 `Recovery` 这两个常用中间件的 Gin 引擎。
        2.  `r.GET("/ping", ...)`：解释这行代码注册了一个 HTTP GET 方法的路由，路径是 `/ping`，并关联了一个处理函数。
        3.  `func(c *gin.Context) {...}`：介绍 `gin.Context` 是 Gin 中最重要的部分，它封装了请求和响应的所有信息。
        4.  `c.JSON(200, ...)`：解释如何使用 `c.JSON` 方法来返回一个状态码为 200 的 JSON 响应。`gin.H` 是 `map[string]interface{}` 的一个快捷方式。
        5.  `r.Run()`：解释它启动了 Web 服务器。

4.  **运行与验证**
    *   指导读者在终端中运行 `go run main.go`。
    *   告诉读者服务器成功启动后会看到的输出。
    *   引导读者打开浏览器或使用 `curl` 命令访问 `http://localhost:8080/ping`。
    *   展示预期的 JSON 返回结果 `{"message":"pong"}`，给读者带来成功的喜悦。

5.  **一个简单的项目结构建议**
    *   解释为什么即使是小项目，也应该有一个基本的结构。
    *   提供一个简单的目录结构示例，用于组织未来的代码：
        ```
        /my-gin-app
        |-- /handlers       # 存放 HTTP 处理函数
        |-- /models         # 存放数据模型
        |-- /routes         # 存放路由定义
        |-- main.go         # 程序入口
        |-- go.mod
        |-- go.sum
        ```
    *   简要说明每个目录的职责，为读者后续扩展应用打下基础。

**总结：**
祝贺读者成功构建并运行了他们的第一个 Gin 应用。鼓励他们继续探索 Gin 更丰富的功能，如路由分组、中间件和请求绑定，并预告下一篇文章将深入这些主题。
