# AI Prompt: 内容贡献模板指南

请你为 `19.模板` 目录撰写一篇说明性的 README 文档。

这篇文档的目的是向所有潜在的贡献者解释此目录中各个模板文件的用途，并指导他们如何使用这些模板来确保新内容的统一性和高质量。

**文档核心内容应包括：**

1.  **引言：为什么需要模板？**
    *   阐明模板对于维护一个大型、协作性强的知识库（如本博客项目）的重要性。
    *   主要优点：保证风格一致、提升内容质量、降低贡献门槛、提高协作效率。

2.  **模板文件详解**
    *   **文章模板 (`01.文章模板.md`)**
        *   **用途**：这是撰写所有新文章的基础框架。
        *   **结构说明**：
            *   `# 标题`: 清晰、吸引人的主标题。
            *   `## 引言`: 简要介绍文章将要讨论的主题及其重要性。
            *   `## 核心概念`: 分点或分段深入讲解关键知识点。
            *   `## 代码示例`: 提供可运行的代码片段，并附上解释。
            *   `## 常见陷阱与最佳实践`: 分享实践经验，帮助读者避免错误。
            *   `## 总结`: 回顾要点，并可能提供进一步的学习资源。
        *   **使用方法**：当撰写新文章时，请复制此文件的内容，并在此结构上进行填充。

    *   **示例代码模板 (`02.示例代码模板.go`)**
        *   **用途**：这是为 `18.示例代码` 目录创建新的 Go 示例程序时应遵循的格式。
        *   **规范说明**：
            *   `package main`: 所有独立运行的示例都应在 `main` 包中。
            *   **文件头注释**: 在文件顶部应有一个简短的注释，说明该示例的功能。
            *   **代码注释**: 对关键或复杂的代码行，应添加行内或行上注释进行解释。
            *   **函数/变量命名**: 遵循 Go 语言的官方命名规范（如驼峰式命名法）。
        *   **使用方法**：创建新的 `.go` 示例文件时，请参考此模板的风格和结构。

3.  **如何贡献**
    *   重申贡献流程：复制模板 -> 填充内容 -> 提交 Pull Request。
    *   鼓励贡献者在开始大量工作前，先提出一个 Issue 来讨论他们的想法。

**总结要求：**

请确保文档内容清晰、直接，具有很强的指导性。让任何一个新来的贡献者都能通过阅读这篇文档，快速理解如何开始他们的第一次内容贡献。
